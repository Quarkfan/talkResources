# AI 工具场景速查表

> **用途**：分享交流时应对"什么工具能解决我的 XXX 问题"类提问
>
> **原则**：按场景分类，不按工具罗列。对方说痛点，你立刻能对应到工具类型。
>
> **工具列举非穷尽、非推荐**。AI 工具迭代极快，此表只反映截至 2025 年中的市场情况。

---

## 使用说明

当对方提出具体痛点时，按以下三步应对：

1. **归类**：这个痛点属于哪个场景类别
2. **匹配**：该类别下的代表性工具
3. **引导**：不直接推荐工具，而是回到方法论框架——"先写到痛点池里，投票决定"

**绝对不要做的事**：当场给出"你应该用 XX 工具"的答案。这违背了方法论的"先验证需求，再选择方案"原则。

## 一、文档/票据/表格处理

### 典型痛点
- "我每天花 2 小时给电子发票查重验真"
- "收到供应商合同要手动录入甲方乙方、金额、到期时间"
- "报销单和发票要对半天"
- "仓库收货单要和采购单一条一条核对"

### 场景抽象
**结构化信息提取**：把非结构化文件（PDF、图片、纸质文件）中的关键信息提取出来，自动填入结构化表格。

### 代表性工具

| 工具 | 类型 | 核心能力 | 适用度 |
|------|------|----------|--------|
| **阿里通义千问**（qwen.aliyun.com） | 国内大模型 | 多模态理解，文档/图片内容提取，支持 API 对接 | ★★★★★ |
| **百度文心一言**（yiyan.baidu.com） | 国内大模型 | 文档解析、表格提取、合同关键信息抽取 | ★★★★★ |
| **腾讯混元**（hunyuan.tencent.com） | 国内大模型 | 文档智能处理，微信生态集成 | ★★★★ |
| **智谱清言**（chatglm.cn） | 国内大模型 | GLM-4 模型，文档理解、长文本处理 | ★★★★ |
| **Kimi**（kimi.moonshot.cn） | 国内大模型 | 长文本处理（200K+），适合大批量文档 | ★★★★★ |
| **飞书智能伙伴** | 国内办公平台 | 文档智能处理、会议摘要、表格自动化 | ★★★★ |
| **钉钉 AI 助理** | 国内办公平台 | 审批单自动处理、合同解析、表格生成 | ★★★★ |
| **合合信息（TextIn）** | 国内 OCR 专精 | 票据 OCR、合同解析、高精度结构化提取 | ★★★★★（票据场景） |
| **腾讯云 OCR** | 国内 OCR API | 发票、营业执照、身份证等专项识别 | ★★★★★（发票场景） |
| **百度 OCR** | 国内 OCR API | 通用文字识别 + 专项票据识别 | ★★★★ |
| **Microsoft Copilot** | 国外 | Excel 公式生成、文档解析、Power Automate 集成 | ★★★★（外企适用） |

### 方法论中的落地方式
IT 用 AI 开发一个轻量 Web 工具：拖入文件 → 自动提取 → 生成 Excel。不需要采购专门的票据系统，1-2 天就能做出可用原型。

## 二、报表/数据分析自动化

### 典型痛点
- "我每天花 3 小时从 ERP 导数据做日报"
- "销售数据、库存数据要手动拼到一起才能出报表"
- "老板每天早上要看昨天的生产数据，我得提前 1 小时整理"
- "每月对账要花一整天"

### 场景抽象
**数据聚合 + 格式化输出**：从多个数据源获取数据，按固定格式整理，定时输出报表。

### 代表性工具

| 工具 | 类型 | 核心能力 | 适用度 |
|------|------|----------|--------|
| **帆软 FineBI / FineReport** | 国内 BI 平台 | 企业级报表、数据可视化、定时推送 | ★★★★（适合深化期） |
| **永洪 BI** | 国内 BI 平台 | 自助分析、拖拽式报表 | ★★★ |
| **阿里云 DataV** | 国内数据可视化 | 大屏展示、实时数据 | ★★★ |
| **观远数据** | 国内 BI 平台 | 零售/快消行业报表模板 | ★★★★ |
| **WPS AI** | 国内办公软件 | Excel 公式自动生成、智能数据分析 | ★★★★★（轻量场景） |
| **通义千问 + Python** | 国内大模型 + 脚本 | 自定义报表逻辑、定时任务 | ★★★★★（IT 开发场景） |
| **Power BI** | 国外 BI 平台 | 数据可视化、定时刷新、Microsoft 生态 | ★★★★（外企适用） |
| **Tableau** | 国外 BI 平台 | 高级数据分析、可视化 | ★★★（偏复杂） |

### 方法论中的落地方式
第一期：IT 用 AI 写一个 Python 脚本 + Web 界面，从 ERP 导出数据自动拼好报表，定时发给相关人员。不用部署整套 BI 系统。

### 典型痛点

## 重要提醒

1. **不要在分享时推荐具体工具**。对方问"我该用什么"时，引导回方法论："先把需求写到痛点池，IT 评估后选择最合适的工具来交付。"

2. **工具只是交付手段**。用 AI 做、用低代码做、用 RPA 做、采购现成的做——方法论不绑定任何工具，只看哪种方式最快解决问题。

3. **工具列表会过时**。这份文档的价值不在于"列出了所有工具"，在于"建立了场景到工具类型的映射关系"。工具会迭代，场景是稳定的。

4. **安全合规优先**。涉及敏感数据的场景（财务、人事），优先考虑本地部署方案（如私有化部署的通义千问、智谱 GLM 等），不要直接调用公有 API。
