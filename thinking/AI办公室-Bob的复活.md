# AI 办公室：Microsoft Bob 的复活

## 结论

Microsoft Bob 的"拟人化办公室"设想，很可能会以 AI agent 的形式复活，但不会以当年那种卡通房间、卡通角色、强制替代桌面的方式复活。

更可能出现的是：

办公室不再只是文件、应用、菜单和流程，而是一组可对话、可委托、可协作、可审计的岗位型 agent。

例如：

| 传统岗位/职能 | AI 时代可能的 agent |
|---|---|
| 行政助理 | 安排会议、订票、整理日程、提醒事项 |
| 财务助理 | 报销初审、发票归档、预算查询 |
| 法务助理 | 合同条款检查、风险提示、模板生成 |
| 项目经理助理 | 跟踪任务、生成周报、催办、识别延期 |
| HR 助理 | 简历初筛、入职流程、政策问答 |
| 数据分析师 | 拉数、解释指标、生成图表和洞察 |
| 运维工程师助理 | 监控告警解释、故障排查建议、工单联动 |

这其实就是 Bob 的愿景去卡通化、去玩具化、企业级化之后的形态。

---

## Bob 当年的核心设想并不荒唐

Microsoft Bob 1995 年推出，试图把 Windows 的文件、程序和设置包装成一个"家庭/办公室式"的图形环境，用房间、门、桌子、角色助手来降低电脑使用门槛。它后来被视为微软著名失败产品之一，评论普遍批评其笨重、幼稚、低效、硬件要求高，销量也远低于预期。

但它背后的几个判断其实很超前：

1. **人不想理解计算机结构**：用户并不天然关心"文件系统、应用程序、窗口、菜单、路径、权限"。
2. **用户更习惯以任务为中心**：例如"帮我写封信""安排一个会议""查一下这个客户的合同"，而不是"打开 Word → 找模板 → 保存到某个目录 → 发邮件"。
3. **软件可以有角色感**：Bob 试图把功能拟人化，让用户通过"助手"完成任务。
4. **界面可以模拟现实组织**：房间、桌面、信箱、日历、管家，本质上是把抽象计算映射成用户熟悉的社会结构。

这些方向在今天看并没有错。错的是：当年的技术能力支撑不了这个设想。

---

## Bob 为什么失败

Bob 的问题不是"拟人化"本身，而是拟人化之后没有真正的智能。

它只是一个包装层：

- 不能真正理解自然语言；
- 不能跨应用完成复杂任务；
- 不能学习用户偏好；
- 不能访问足够多的上下文；
- 不能承担实际工作责任；
- 还增加了操作路径和视觉负担。

所以用户感受到的不是"智能助理"，而是：

**一个挡在我和电脑之间的玩具界面。**

这和后来 Clippy 的问题类似：角色很显眼，但能力不足；出现频率很高，但帮助有限。

---

## AI 时代让 Bob 的核心设想重新成立

今天变化很大。

Microsoft 365 Copilot 已经支持面向特定场景的 agents，包括 declarative agents 和 custom engine agents；前者适合在 Microsoft 365 Copilot 内做聚焦场景，后者适合复杂工作流和高级集成。OpenAI 也已经把 agent 能力推进到"可以使用自己的计算机完成复杂任务"的方向，例如 ChatGPT agent 和此前的 Operator / Computer-Using Agent，目标就是从"回答问题"走向"执行任务"。

这意味着 Bob 当年缺失的几个条件现在开始具备：

| Bob 时代缺失的能力 | AI 时代的补足 |
|---|---|
| 理解自然语言 | LLM 可以理解复杂意图 |
| 任务规划 | Agent 可以拆解步骤 |
| 跨软件操作 | API、RPA、浏览器控制、GUI agent |
| 个性化 | 长上下文、记忆、企业知识库 |
| 组织协作 | 多 agent、权限、流程、审计 |
| 内容生成 | 文档、邮件、表格、PPT、代码生成 |

所以，Bob 的旧愿景可以重述为：

**用户不再操作软件，而是管理一组有职责、有权限、有上下文的数字员工。**

---

## "办公室里的各个岗位都是 agent"是很自然的演进

这个方向很可能成立，尤其是在企业内部系统中。

传统企业软件是这样的：

> 人 → 打开多个系统 → 查询数据 → 判断 → 填表 → 发邮件 → 跟进

Agent 化之后更像这样：

> 人 → 提出目标 → 相关岗位 agent 协作 → 人类审批关键节点 → 系统执行

例如一个"项目办公室"可以由这些 agent 组成：

- **CEO Agent**：看战略目标、风险、经营指标
- **PM Agent**：拆任务、跟进进度、生成周报
- **Finance Agent**：预算、成本、报销、现金流
- **Legal Agent**：合同、合规、风险条款
- **Sales Agent**：客户跟进、报价、CRM 维护
- **Ops Agent**：故障、工单、SLA、资源调度
- **Knowledge Agent**：制度、文档、经验库问答

用户不是在 Word、Excel、ERP、OA、CRM 之间跳转，而是在和不同职责的 agent 协作。

这就是 Bob 的"房间和角色"概念在企业 AI 时代的复活版。

---

## 但它不应该长得像 Bob

真正可能成功的形态，不是把屏幕变成一个卡通办公室。

更合理的形态是三类界面叠加：

### 1. 对话式入口

用户说：

> "让财务 agent 看一下这个项目预算有没有异常。"

或者：

> "让法务 agent 帮我审一下这份供应商合同。"

这是最接近 Bob 的部分：用自然语言调用"角色"。

### 2. 工作台式界面

不是卡通房间，而是一个 agent 工作台：

| Agent | 状态 | 正在处理 | 需要我确认 |
|---|---|---|---|
| 财务 Agent | 运行中 | 审核 12 张发票 | 2 个异常 |
| 法务 Agent | 等待确认 | 合同风险标注 | 需要审批 |
| PM Agent | 已完成 | 本周项目周报 | 可发送 |
| 运维 Agent | 告警中 | 水压异常分析 | 建议派单 |

这比"卡通办公室"更适合企业场景。

### 3. 嵌入式 agent

Agent 不一定是一个独立角色，也可以嵌入到现有软件里：

- 在 Word 里是合同审查 agent；
- 在 Excel 里是数据分析 agent；
- 在 Teams/飞书/钉钉里是会议和任务 agent；
- 在 ERP 里是财务和采购 agent；
- 在 GIS/SCADA/IoT 平台里是运维 agent。

这比重新发明一个桌面系统更现实。

## 最大机会：岗位级 agent，而不是人格化宠物

Bob 的问题之一是"角色先行，能力不足"。

AI 时代应该反过来：

**不是先设计一个可爱的角色，而是先定义一个清晰岗位。**

一个好的企业 agent 应该像岗位说明书一样定义：

```
名称：合同审查 Agent
职责：识别合同风险、对比模板、生成修改建议
权限：可读取合同库、供应商资料、历史审查意见
限制：不能自动签署合同，不能绕过法务审批
输入：合同文档、交易背景、供应商信息
输出：风险清单、修改建议、审批摘要
审计：记录每次判断依据和引用来源
```

这就比"我是一个会说话的小狗助手"可靠得多。

---

## 最大风险：企业里不能有"黑箱员工"

如果办公室里的岗位都是 agent，最关键的问题不是 UI，而是治理。

需要解决：

1. **权限边界**：财务 agent 能不能看 HR 数据？销售 agent 能不能看利润率？运维 agent 能不能直接重启设备？
2. **责任归属**：Agent 做错了，谁负责？使用者、部门负责人、IT、供应商，还是模型厂商？
3. **审批机制**：哪些事情可以自动执行，哪些必须人工确认？
4. **可审计性**：Agent 为什么这么判断？引用了哪些数据？执行了哪些动作？
5. **组织适配**：Agent 不能只是"聪明"，还要符合企业流程、制度和权限体系。
6. **用户控制感**：微软最近允许 Office 用户移除干扰性的 Copilot 浮动按钮，说明即使是 AI 功能，如果侵入感太强，也会引发反感。这正是 Bob/Clippy 留下的教训：助手不能挡路。

## 对未来交互模式的判断

我认为会出现三个阶段。

### 阶段一：单点 Copilot

每个软件里都有 AI 助手。

- Word 里帮你写；
- Excel 里帮你分析；
- 邮件里帮你回复；
- IM 里帮你总结。

这是现在已经发生的阶段。

### 阶段二：岗位型 Agent

每个业务职能有自己的 agent。

- 财务 agent；
- 法务 agent；
- 项目 agent；
- 采购 agent；
- 运维 agent；
- 客服 agent。

这会成为企业 AI 落地的主战场。

### 阶段三：组织级 Agent Network

多个 agent 之间自动协作，人类变成目标设定者和关键审批者。

例如你说：

> "下个月完成 A 项目的交付准备。"

系统自动分派：

- PM Agent：拆交付计划
- 采购 Agent：确认设备到货
- 财务 Agent：核对预算
- 法务 Agent：检查合同交付条款
- 运维 Agent：准备部署资源
- 客服 Agent：准备客户培训材料

最后给你一个：

```
交付准备状态：78%
主要风险：设备到货延迟、客户验收标准未确认
需要你审批：追加预算 12 万元
建议动作：今天联系供应商确认交期
```

这才是真正意义上的"AI 办公室"。

---

## 所以，Bob 会复活吗？

会，但不是以 Bob 的外形复活，而是以它的底层隐喻复活。

| Bob 的元素 | AI 时代是否会复活 | 形式 |
|---|---|---|
| 拟人化助手 | 会 | 岗位型 agent |
| 房间/办公室隐喻 | 部分会 | 工作台、组织视图、流程视图 |
| 卡通角色 | 小范围会 | 教育、儿童、陪伴、轻办公 |
| 替代桌面系统 | 不太可能 | 现有软件中嵌入 agent |
| 自然语言操作 | 会 | 主入口之一 |
| 软件替你办事 | 会 | Agent + API + 审批流 |
| 主动提醒 | 会 | 但必须可控、低打扰 |
| 全自动办公 | 有限 | 高风险动作仍需人类审批 |

---

## 一个更准确的名字

它不应该叫"新 Bob"。

更准确的概念可能是：

- Agentic Office
- 数字岗位系统
- AI 组织操作系统
- 企业 Agent 工作台
- Role-based AI Interface

Microsoft Bob 是"把电脑伪装成办公室"。

AI agent 是"把办公室抽象成一组可委托的数字岗位"。

这两者表面相似，但本质上后者更有机会成功。
